User Interface

  • The chaiproject UI is based on mainly JQuery and Twitter Bootstrap. All the other features are called "plugins" that are loaded on demand.
  • All plugins are accessed via a hash (fragment). For example signin can be called by adding #signin to the addressbar

Standard Interface Features:

  1. Dynamic Pages: Pages can be created on the fly and loaded via ajax. Current page has content and active css class on it.
  2. Toolbar: Standard Twitter Bootstrap Toolbar with buttons for login, admin and user settings
  3. Admin Pages: List of pages, users, files. Adding and deleting
  4. File Upload: Dialog (modal) for uploading a file
  5. Page Editing: Click on Admin->Edit on any page

Types of Pages

Pages can be of 2 types

  1. Virtual Pages (stored primarily in the db, also stored as files for search bots)
  2. Plugins (stored only as files)

Adding new Pages

  1. To add a new Virtual Page, just go to Admin-> Pages and "Add". You can either edit the HTML or markdown. (Markdown to html, courtesy of showdown.js)

  2. To add a new plugin, just make a new html file preferably in the "plugins" folder of your root directory and register the plugin in your index_template.html by add the following js line.

$.extend($._plugins, {"myplugin": {"path":"plugins"}});

This will ensure that this file will be loaded when #myplugin is called
